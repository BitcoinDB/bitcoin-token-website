BitcoinToken Install Guide

VERSION v0.0.1-alpha.5

This page contains instructions for how to install BitcoinWallet, BitcoinDb, and BitcoinToken in a browser and in node.

BitcoinWallet

The easiest way to get started is to use BitcoinWallet in node.

Node

You need node.js and npm installed. Then create a file index.js

          const Bitcoin = require('bitcointoken')
          const wallet = new Bitcoin.Wallet()
          
          console.log(`
          \tprivateKey: ${wallet.getPrivateKey()}
          \taddress: ${wallet.getAddress()}
          `)

Type node --experimental-repl-await index.js into a terminal window. If everything worked, you should see output similar to this:

          privateKey: da31ca9a791c6214069b5e87f8080ba69d94336dee54f0421c4019d9e41ddca8
          address: mwZd1bgRYhxY4JLyx1sdEGBYFZnXVDvgmp

Browser

You need node.js and npm installed. We recommend to use Parcel to bundle the files for the browser.

          npm install -g parcel-bundler

Initialize an empty project

          npm init -y

Create a file called index.js with the following content

          const Bitcoin = require('bitcointoken')
          const wallet = new Bitcoin.Wallet()

          document.body.innerHTML = 
           `<h1>Your Wallet</h1>
            privateKey: ${wallet.getPrivateKey()}<br>
            address: ${wallet.getAddress()}`

Then create a file called index.html

          <!doctype html>

          <html lang='en'>
            <body>
              <script src="./index.js"></script>
            </body>
          </html>

Start a development server using

          parcel index.html

To see the rendered website open http://localhost:1234/ in a browser. You can use Parcel to bundle your files for production.

BitcoinDb

BitcoinDb stores data in specialized (non-standard) transactions. As non-standard transactions are not propagated by all Bitcoin nodes, we use a technique called P2SH to make sure that transactions are propagated immediately. However, to make things work you need to run an additional server called the Bitcoin Non-Standard Server.

In this section we explain how to build and run a non-standard server. Once the non-standard server is running, you can use BitcoinDb in the same way you can use BitcoinWallet.

Build and run a non-standard server

We recommend installing Docker and Docker Compose. (Instruction for how to build and run the non-standard server without Docker are in the Github repo). If you have Docker installed, you can build and run the non-standard server by typing the following into a terminal window:

          git clone https://github.com/BitcoinDB/bitcoin-non-standard-server.git
          cd bitcoin-non-standard-server/
          docker-compose up

The non-standard server is now running on localhost:3000. Make sure that the non-standard server is running when using BitcoinDb. BitcoinDb will connect to the server automatically.
